# Harubeori Roadmap

Harubeori is an early-stage Korean digital diary project.  
This roadmap describes the planned direction for improving the app as an open-source project.

## v0.4.1 - Current Prototype

Current status:

- Korean-first daily diary interface
- Date-based daily record structure
- Three-step daily flow: 비움 · 세움 · 엮음
- Mobile web app deployment through GitHub Pages
- Local browser-based storage
- Early user testing with friends and family
- Feedback collected on photo memory, backup, and mobile usability

## v0.4.2 - Mobile Writing Improvement

Planned:

- Improve mobile input stability
- Reduce layout jumping when typing
- Improve button spacing on mobile
- Improve date selection behavior
- Improve save-state feedback
- Test on Android Chrome and iOS Safari

## v0.5.0 - Photo Memory Update

Planned:

- Add date-based photo upload
- Limit daily photo storage to 1–2 meaningful photos
- Add memo field for each uploaded photo
- Add image preview layout
- Add fallback UI when no photo is uploaded
- Improve storage handling for compressed images

## v0.6.0 - Backup and Restore

Planned:

- Export all text records as JSON
- Export photo memories and photo memos
- Import backup JSON from another device
- Add warning for private photo data
- Add beginner-friendly backup guide
- Improve backup file size handling

## v0.7.0 - Open Source Readiness

Planned:

- Improve README
- Add screenshots
- Add contribution guide
- Add issue templates
- Add beginner-friendly setup guide
- Add code comments
- Refactor code into clearer modules

## Long-term Ideas

- Optional Google Drive backup
- Optional Notion export
- Family memory mode
- Printable daily summary
- Offline-first PWA support
- Korean beginner developer learning guide
- React or Vite-based refactoring

# Changelog

All notable changes to Harubeori will be documented in this file.

## v0.4.1

### Added

- Restored the main Harubeori interface after the v0.4 prototype issue
- Added support concept for full backup and restore
- Added text-only backup option
- Added photo memory concept
- Added photo memo concept
- Preserved the core structure: 첫장 / 비움 / 세움 / 엮음 / 기록

### Changed

- Improved the project direction toward an open-source digital diary
- Refined the user flow based on early feedback
- Improved mobile-first project positioning

### Known Issues

- Photo storage still needs more testing
- Mobile input behavior needs continued improvement
- Backup and restore flow needs better user guidance
- The codebase needs refactoring for easier contribution

## v0.4.0

### Added

- Experimental backup and restore concept

### Known Issues

- A simplified fallback UI was accidentally deployed
- The main Harubeori design was not preserved correctly

## v0.3.0

### Added

- Date-based photo memory concept
- Daily photo limit concept
- Photo memo concept

## v0.2.0

### Fixed

- Reduced mobile input jumping while typing
- Improved save behavior during writing

## v0.1.0

### Added

- Initial Harubeori digital diary prototype
- Korean-first diary interface
- Date-based writing
- 비움 · 세움 · 엮음 structure
- GitHub Pages deployment
